# Device Types

Device categories matter because they define what a system can do reliably, what it costs to operate, and how much integration burden lands on the project team. This section clarifies job boundaries between hardware classes.

## Key questions

<Steps>
	1. Does the site need translation, control, local logic, or just reliable data transport?
	2. What uptime, maintenance, and power assumptions does the hardware need to survive?
	3. How much software ownership is the operating team prepared to absorb?
	4. Which category minimizes avoidable complexity while still meeting the site objective?
</Steps>
