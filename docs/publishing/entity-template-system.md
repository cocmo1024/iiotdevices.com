# IIoT Devices Template System

This repository is designed for AI-operated publishing. New pages should follow the shared frontmatter contract and the matching outline for the page type.

## Shared frontmatter

Use these fields on new pages:

- `title`
- `description`
- `referenceType`
- `adProfile`
- `commercialIntent`
- `reviewCadence`
- `primaryKeyword`
- `searchIntent`
- `decisionStage`
- `targetRoles`
- `contentStatus`

## Vendor template

Use for Siemens, Phoenix Contact, Advantech, Moxa, and similar portfolio pages.

Sections:

1. Why the vendor matters
2. Where the portfolio fits best
3. Strengths in real buying contexts
4. Limits or watchouts
5. Where to compare next

## Product family template

Use for gateways, RTUs, remote I/O, switches, edge computers, and related hardware categories.

Sections:

1. What the category is for
2. Where it fits in the architecture
3. When the category is overkill or wrong
4. Buying signals
5. Related protocols, vendors, or applications

## Protocol template

Use for Modbus, MQTT, OPC UA, serial bridging, and translation topics.

Sections:

1. Which layer this protocol belongs to
2. Typical fit
3. Common architecture mistake
4. Translation or device implications
5. Compare next

## Application template

Use for retrofit, remote monitoring, machine connectivity, and similar deployment contexts.

Sections:

1. Why the application matters commercially
2. Site constraints and pressures
3. Architecture implications
4. Product-family and protocol fit
5. Common buying mistakes

## Comparison template

Use for shortlist-stage pages with strong purchase intent.

Sections:

1. Comparison scope
2. Decision criteria
3. Where each option fits
4. Failure cases
5. Next-step references
